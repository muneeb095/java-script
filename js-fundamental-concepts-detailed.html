JavaScript ke Buniyadi Concepts (Detailed Guide)

1. Data Types (Data ki Qismein)

Sochlein ke aapke paas alag alag qisam ke dabbe (containers) hain. Ek mein aap sirf "likhai" rakh sakte hain, ek mein "adad", aur ek mein sirf "Haan" ya "Na" ka jawab.

Primitive Types (Buniyadi - Sirf Ek Cheez Store Karne Walay)

  • String: Koi bhi text. Iski pehchan hai ke yeh hamesha quotes "" ya '' ke andar hota hai.
    • var fruitName = "Apple";
  • Number: Koi bhi adad (chahe poora ho ya ishariya wala). Ismein quotes nahi hotay.
    • var price = 150;
    var tax = 0.5;
  • Boolean: Sirf do hi qeemtein ho sakti hain: true (sach/haan) ya false (jhoot/na). Yeh `if` statements mein bohat istemal hota hai.
    • var isAdult = true;

Non-Primitive Types (Advance - Ek se Zyada Cheezein Store Karne Walay)

  • Object: Ek aisi cheez jismein multiple khasiyatein (properties) hoti hain. Jaise ek "Car" ka object, jiska color, model, aur speed alag alag properties hain.
    • var car = {
        color: "Red",
        model: 2024
    };
  • Array: Ek jaisi cheezon ki ek list. Jaise students ke naam ya shopping ki list.
    • var fruits = ["Apple", "Banana", "Orange"];

2. Variables (var, let, const)

Yeh wohi data ke containers hain. Naye JavaScript (ES6) mein inko banane ke 3 tareeqe hain.

  • var: Yeh purana tareeqa hai. Iski value baad mein badal sakti hai. Iska scope thora ajeeb hai (function scope), isliye ab isay kam istemal karte hain.
  • let: Yeh naya aur behtar tareeqa hai. Iski value bhi baad mein badal sakti hai.
    • let myAge = 25;
    myAge = 26; // Theek hai, value badal gayi.
  • const: "Constant" se nikla hai. Iski value ek baar set ho jaye to dobara badal nahi sakti.
    • const myCountry = "Pakistan";
    myCountry = "India"; // Error! Aap const ki value nahi badal sakte.

Rule of Thumb: Hamesha const istemal karein. Jab aapko pata ho ke value baad mein badalni hai, tab let istemal karein. var ko naye code mein bhool jayen.

3. Scope (Variable ki Puhnch ya Daira-e-Kaar)

Scope ka matlab hai ke ek variable aapke code ke kis hissay mein "zinda" ya "available" hai.

  • Global Scope: Jo variable kisi bhi function ya block {} ke bahar banaya jaye, woh "public property" jaisa hai. Usay code mein kahin se bhi istemal kiya ja sakta hai.
    • var globalMessage = "Hello everyone!"; // Yeh Global hai
  • Function Scope: Agar aap var ko kisi function ke andar banate hain, to woh sirf us function ke andar hi zinda rehta hai. Function ke bahar usay koi nahi janta.
  • Block Scope {}: Yeh let aur const ki khasiyat hai. Agar aap inhein kisi bhi curly braces {} (jaise if, for) ke andar banate hain, to woh sirf unhi braces ke andar tak zinda rehte hain.
if (true) {
    let blockMessage = "Sirf is block mein hoon"; // Yeh Block-scoped hai
}
// console.log(blockMessage); // Error! Kyunki blockMessage bahar zinda nahi hai.

4. Functions, Methods, aur Properties

Aasan misal: Ek insaan (Object) ko sochen.

  • Property (Khasiyat): Insaan ka naam, umar, rang. Yeh uski khasiyatein hain. Yeh values hain.
    • var person = { name: "Ali" };
    console.log(person.name); // 'name' ek property hai
  • Method (Kaam/Amal): Insaan ka chalna(), bolna(), khana(). Yeh woh kaam hain jo woh karta hai. Inke aage hamesha parentheses () hotay hain.
    • console.log("Hello"); // 'log()' ek method hai jo 'console' object se jura hai.
  • Function: Ek azaad "kaam" jo kisi object se nahi jura. Yeh ek recipe ki tarah hai, jise aap naam dekar jab chahein istemal kar sakte hain.
    • alert("Hello"); // 'alert()' ek predefined function hai.

5. Keywords (JavaScript ke Reserve Alfaz)

Yeh woh khaas alfaz hain jinka JavaScript mein pehle se ek matlab hai. Aap inhein variable ke naam ke taur par istemal nahi kar sakte.

Misalein: if, else, for, var, let, const, function, return, true, false, while, switch, etc.

var for = "Loop"; // Error! 'for' ek keyword hai.
