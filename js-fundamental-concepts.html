1. Data Types (Data ki Qismein)

Har variable mein jo data store hota hai, uski ek qisam hoti hai. Yeh do baray groups mein taqseem hoti hain:

  • Primitive Types (Buniyadi Qismein): Yeh sirf ek (single) value store karti hain.
    • String: Text, jo hamesha quotes "" mein hota hai. e.g., "Hello World"
    • Number: Koi bhi adad, bina quotes ke. e.g., 100, 3.14
    • Boolean: Sirf do values: true (sach) ya false (jhoot).
    • Undefined: Jab variable banaya gaya ho lekin usay koi value na di gayi ho.
    • Null: Jab hum khud se kehte hain ke ismein "kuch nahi" hai.
  • Non-Primitive (or Reference) Types: Yeh ek se zyada (multiple) values ya complex data store karti hain.
    • Object: Key-value pairs ka collection. e.g., { name: "Ali", age: 25 }
    • Array: Ek list, jismein multiple values store hoti hain. e.g., ["apple", "banana", "orange"]

2. JavaScript Kaise Kaam Karti Hai? (V8 Engine)

Aap jo code likhte hain, computer usay direct nahi samajhta. Usay machine language (0s and 1s) mein badalna parta hai.

  • Translator (Interpreter/Compiler): Yeh woh program hai jo aapke code ko machine language mein badalta hai.
  • V8 Engine: Yeh Google Chrome ka JavaScript engine (translator) hai. Yeh bohat tez hai kyunki yeh **Just-In-Time (JIT) compilation** istemal karta hai, jo interpreter aur compiler dono ki khoobiyon ko milata hai. Aasan lafzon mein, yeh aapke code ko run hone se pehle hi optimize kar leta hai taake woh tezi se chale.

3. Static vs Dynamic

Is context mein, iska matlab hai ke website ka content kaise badalta hai.

  • Static: HTML aur CSS se bani website jiska content tab tak nahi badalta jab tak aap code ko khud na badlein.
  • Dynamic: JavaScript ke zariye hum website ko dynamic banate hain. Matlab, user ke actions (click, scroll) par content badal sakta hai, bina page reload kiye. Yeh sab **DOM (Document Object Model)** ke zariye hota hai.

4. Variables (var, let, const)

Yeh data store karne ke containers hain. Teen tarah ke hotay hain:

  • var: Yeh purana tareeqa hai. Iski value badal sakti hai aur yeh **function-scoped** hota hai.
  • let: Yeh naya aur behtar tareeqa hai. Iski value badal sakti hai aur yeh **block-scoped** hota hai.
  • const: "Constant" se nikla hai. Iski value ek baar set ho jaye to **dobara badal nahi sakti**. Yeh bhi **block-scoped** hota hai.

Achi Practice: Ab naye code mein zyada tar let aur const istemal karein. var ko kam se kam istemal karein.

5. Scope (Variable ki Puhnch)

Scope ka matlab hai ke ek variable aapke code ke kis hissay mein "zinda" ya "available" hai.

  • Global Scope: Jo variable kisi bhi function ya block ke bahar banaya jaye, woh har jagah available hota hai.
  • Function Scope: Jo variable var ke saath kisi function ke andar banaya jaye, woh sirf us function ke andar hi istemal ho sakta hai.
  • Block Scope {}: Jo variable let ya const ke saath kisi curly braces {} (jaise if, for) ke andar banaya jaye, woh sirf us block ke andar hi istemal ho sakta hai.

6. Functions, Methods, aur Properties

Inmein thora sa farq hai:

  • Function: Ek code ka block jise hum naam dekar baar baar chala sakte hain. e.g., alert(), prompt().
  • Method: Ek function jo kisi object ke saath jura ho. e.g., console.log(). Yahan console ek object hai aur log uska method hai.
  • Property: Ek object ke saath juri hui koi value ya khasiyat. e.g., array.length. Yahan array ek object hai aur length uski property (khasiyat) hai.
var cars = ["Saab", "Volvo", "BMW"];
cars.length; // Yeh 'property' hai (jawab 3 aayega)
console.log(cars); // Yeh 'method' hai (array ko console mein print karega)

7. Keywords (Khaas Alfaz)

Yeh JavaScript ke apne reserve alfaz hain jinka khaas matlab hota hai aur aap inhein variable ke naam ke taur par istemal nahi kar sakte.

e.g., if, else, for, var, function, return, etc.

8. Nested `if` (if ke andar if)

Jab aap ek if statement ke andar ek aur if statement likhte hain to isay "nesting" kehte hain. Yeh complex conditions check karne ke kaam aata hai.
